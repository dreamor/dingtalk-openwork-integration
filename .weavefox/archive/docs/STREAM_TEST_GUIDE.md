## 📋 钉钉后台配置步骤

### 1. 进入钉钉开发者平台
访问：https://open.dingtalk.com/

### 2. 创建/编辑企业内部应用
- 登录 → 控制台 → 企业内部开发
- 创建新应用或选择现有应用

### 3. 添加机器人能力（Stream 模式）
1. 点击 **应用能力** → **添加应用能力**
2. 选择 **机器人**
3. **关键步骤**：选择 **Stream 模式**（不是 Webhook 模式）
4. 填写机器人信息：
   - 机器人名称：OpenWork 助手
   - 机器人头像：（可选）
   - 简介：（可选）
5. 点击 **发布**

### 4. 获取凭证
在应用基本信息页面，获取：
- **ClientID** (即 AppKey)
- **ClientSecret** (即 AppSecret)

### 5. 配置 .env 文件
```bash
DINGTALK_APP_KEY=你的 ClientID
DINGTALK_APP_SECRET=你的 ClientSecret
STREAM_ENABLED=true
```

> ⚠️ **重要**：Stream 模式**无需配置回调 URL**！

## 🔍 故障排查

### Stream 连接失败

**错误现象**：
```
ERROR Error: connect EBADF
```

**解决方案**：
1. 检查网络连接
2. 确认 AppKey 和 AppSecret 正确
3. 确认应用已发布
4. 确认机器人选择了 Stream 模式

### 收不到消息

**检查清单**：
- [ ] 机器人已添加到聊天
- [ ] 机器人选择了 Stream 模式
- [ ] 服务正在运行
- [ ] AppKey 和 AppSecret 配置正确
- [ ] 网络连接正常

### 查看日志

```bash
# 重启服务查看详细日志
npm start

# 观察日志输出
# 应该看到：
# 🌊 启动 Stream 模式
# 📡 Stream 模式服务已初始化
# 🚀 正在连接钉钉 Stream 服务...
# ✅ Stream 连接已建立
```
