# 钉钉 Stream 模式配置指南

## ✅ 当前状态

- **Stream 连接**: ✅ 已成功建立
- **WebSocket 地址**: `wss://wss-open-connection.dingtalk.com:443/connect`
- **订阅事件**: `/v1.0/im/bot/messages/get`
- **心跳检测**: ✅ 正常（每 8 秒）

## ⚠️ 需要在钉钉开发者后台完成的配置

### 1. 进入钉钉开发者平台

访问：https://open.dingtalk.com/

### 2. 找到你的应用

- 进入「控制台」
- 选择你的应用（AppKey: `dingetmdkk1u75ilvun7`）

### 3. 配置 Stream 模式

#### 方式一：在「开发管理」中配置

1. 进入「开发管理」→「事件订阅」
2. 选择 **Stream 模式**（而非 HTTP 回调模式）
3. 添加订阅事件：
   - **事件类型**: `im`
   - **事件名称**: `机器人消息`
   - **事件标识**: `/v1.0/im/bot/messages/get`

#### 方式二：在「机器人」配置中

1. 进入「机器人」→「功能设置」
2. 找到 **消息接收模式**
3. 选择 **Stream 模式**
4. 确认订阅了机器人消息事件

### 4. 配置机器人权限

确保机器人有以下权限：

- ✅ **个人单聊**: 允许用户私聊机器人
- ✅ **群聊**: 允许机器人在群聊中被@
- ✅ **消息类型**: 支持文本消息

### 5. 验证配置

配置完成后：

1. 重启应用：`npm run dev`
2. 在钉钉中向机器人发送消息
3. 观察日志输出

## 🔍 故障排查

### 问题：连接成功但收不到消息

**可能原因：**

1. **钉钉后台未配置 Stream 模式**
   - 解决：在钉钉开发者后台切换到 Stream 模式

2. **订阅事件不正确**
   - 解决：确认订阅了 `/v1.0/im/bot/messages/get` 事件

3. **机器人权限不足**
   - 解决：检查机器人是否有消息接收权限

4. **消息类型不匹配**
   - 解决：当前只处理文本消息

### 问题：连接失败

**检查项：**

1. AppKey 和 AppSecret 是否正确
2. 网络是否通畅
3. 钉钉应用是否已启用
