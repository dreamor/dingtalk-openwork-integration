## ADDED Requirements

### Requirement: Open Code 消息识别

系统 MUST 识别用户通过钉钉发送的 Open Code 相关消息，并将请求路由到相应的处理器。

#### Scenario: 检测代码生成请求

- **WHEN** 钉钉消息包含以下特征：
  - 消息以 "生成代码"、"写一段"、"创建一个" 等开头
  - 消息包含 "用 [语言] 写"、"使用 Python" 等语言指定
  - 消息包含 "函数"、"类"、"组件" 等编程术语
- **THEN** 系统将消息标记为 Open Code 代码生成请求
- **AND** 提取关键参数（编程语言、功能描述）

#### Scenario: 检测代码审查请求

- **WHEN** 钉钉消息包含以下特征：
  - 消息以 "审查"、"检查"、"看看" 等开头
  - 消息包含代码内容或文件路径
  - 消息提及 "问题"、"优化"、"安全" 等审查相关词汇
- **THEN** 系统将消息标记为 Open Code 代码审查请求
- **AND** 提取代码内容或文件路径

#### Scenario: 检测文件操作请求

- **WHEN** 钉钉消息包含以下特征：
  - 消息包含 "创建文件"、"新建文件"、"读取文件" 等操作词
  - 消息包含文件路径（相对路径或绝对路径）
  - 消息包含文件操作描述
- **THEN** 系统将消息标记为 Open Code 文件操作请求
- **AND** 提取操作类型和目标文件路径

#### Scenario: 检测多轮对话请求

- **WHEN** 钉钉消息包含以下特征：
  - 消息引用之前的代码或对话（"刚才那段"、"之前那个"）
  - 消息包含 "继续"、"完善"、"修改" 等继续词
  - 存在可用的对话历史
- **THEN** 系统将消息标记为 Open Code 多轮对话请求
- **AND** 从对话历史中加载相关上下文

### Requirement: Open Code 消息格式支持

系统 MUST 支持在钉钉消息中发送和接收 Open Code 相关的各种格式内容。

#### Scenario: 接收代码片段

- **WHEN** 钉钉消息包含代码块（使用 ``` 标记的代码）
- **THEN** 系统提取代码块内容
- **AND** 保留代码格式和缩进
- **AND** 去除代码块标记

#### Scenario: 发送代码生成结果

- **WHEN** Open Code 成功生成代码
- **THEN** 系统通过钉钉消息返回结果
- **AND** 使用 Markdown 代码块格式（``` 语言标识）
- **AND** 添加代码功能说明

#### Scenario: 发送代码审查报告

- **WHEN** Open Code 完成代码审查
- **THEN** 系统通过钉钉消息返回审查结果
- **AND** 使用 Markdown 格式，包含问题列表和改进建议
- **AND** 使用 Emoji 辅助标识问题严重程度

#### Scenario: 发送错误消息

- **WHEN** Open Code 执行失败
- **THEN** 系统通过钉钉消息返回错误信息
- **AND** 使用友好且可理解的表述
- **AND** 避免暴露技术细节和敏感信息

### Requirement: Open Code 交互控制

系统 MUST 提供必要的交互控制机制，确保 Open Code 功能的良好用户体验。

#### Scenario: 执行确认

- **WHEN** 用户请求执行可能产生副作用的操作（如文件修改、代码执行）
- **THEN** 系统发送确认消息询问用户
- **AND** 等待用户明确确认后才执行
- **AND** 设置合理的超时时间

#### Scenario: 进度反馈

- **WHEN** Open Code 命令执行时间较长
- **THEN** 系统发送进度更新消息
- **AND** 包含当前步骤说明
- **AND** 避免每条消息都发送更新（建议每 5-10 秒）

#### Scenario: 中断执行

- **WHEN** 用户发送 "停止"、"取消"、"中断" 等命令
- **THEN** 系统终止正在执行的 Open Code 命令
- **AND** 发送执行被中断的通知
- **AND** 清理相关资源

#### Scenario: 帮助信息

- **WHEN** 用户请求 Open Code 相关的帮助
- **THEN** 系统发送 Open Code 功能说明和使用示例
- **AND** 列出可用的命令和参数
- **AND** 提供使用模板

---

### Requirement: Open Code 对话历史管理

系统 MUST 管理 Open Code 相关的对话历史，支持多轮交互。

#### Scenario: 保存对话上下文

- **WHEN** Open Code 命令执行完成
- **THEN** 系统保存相关上下文信息
- **AND** 保存用户请求、生成的代码、审查结果等
- **AND** 设置上下文保留时间限制（建议 30 分钟）

#### Scenario: 恢复对话上下文

- **WHEN** 用户发送继续性请求（"继续完善"）
- **THEN** 系统从对话历史中恢复相关上下文
- **AND** 加载最近的相关对话记录
- **AND** 将上下文传递给 Open Code CLI

#### Scenario: 上下文长度限制

- **WHEN** 对话历史过长
- **THEN** 系统截取最近的相关部分
- **AND** 保持上下文的关键信息完整性
- **AND** 通知用户上下文已被精简
