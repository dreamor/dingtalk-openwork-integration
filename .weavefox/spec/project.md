# 项目上下文

## 目的

实现一个基于钉钉集成的自动化系统，通过钉钉 Channel 与本地的 open work CLI 进行通信，用户可以通过钉钉消息控制 open work CLI 执行各种本地命令操作。

## 系统架构

```
钉钉 Channel → Gateway(local) → LLM → Tools(本地命令) → 回复
```

### 架构组件说明

1. **钉钉 Channel**: 用户通过钉钉发送消息和命令
2. **Gateway(local)**: 本地网关服务，接收钉钉消息并转发
3. **LLM**: 大语言模型，解析用户意图并生成响应
4. **Tools(本地命令)**: 执行实际的本地 CLI 命令
5. **回复**: 将执行结果通过钉钉返回给用户

## 技术栈

- **运行时**: Node.js / TypeScript
- **钉钉集成**: 钉钉开放平台 API
- **LLM**: 大语言模型接口
- **本地命令执行**: shell 命令执行模块

## 功能规划

1. **钉钉消息接收**: 配置钉钉机器人接收用户消息
2. **消息解析**: LLM 解析用户意图和命令
3. **命令执行**: 安全地执行本地 CLI 命令
4. **结果返回**: 将执行结果格式化并返回到钉钉
5. **权限控制**: 确保只有授权用户可以执行命令

## 安全考虑

- 命令白名单机制
- 用户身份验证
- 命令执行权限控制
- 敏感操作确认机制
